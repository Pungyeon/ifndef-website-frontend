Overview

Architecture Pretty icons

The workflow of a user request for this site, is mapped below. A user will request www.pungy.net (which is hosted on Route53... that's not in the diagram though). This request will be routed to AWS CloudFront, which will forward the request to an S3 bucket, which hosts the minimised javascript UI.

The javascript UI will be sent to the user and will be invoked on the clients machine. The UI will then request information from the backend. These requests will go to AWS API Gateway, which in turn will forward (and authorize) the request to AWS Lambda, which will invoke a NodeJS code to retrieve the data from DynamoDB.

So, perhaps a little complicated for hosting a simple website. However, once setup, there is little to no maintanence and it's favourably cheap to host.

Frontend

Angular User Interface

What is Angular?

Angular is an open-source javascript framework for developing graphical user interfaces. Angular is Google backed and is technically a 2.0 version of the popular AngularJS framework. To learn more about Angular, I recommend visiting the official site: https://angular.io/

Why Angular?

So, there are a lot of different frameworks to choose from. Angular, React and Vue are currently the three most popular frameworks to choose from. Choosing a popular framework typically comes with the side-effect of better community support, making it easier to acquire help when troubleshooting. I chose Angular over React, simply because I prefer the syntax and structure og Angular over React, especially when it comes to the dependency injection model of Angular vs. React. I also really like the opinionatedness of Angular, which I feel, helps ensure consistency between projects, both in writing style and structure.

Vue was another option, which looks super promising, but at the time of writing this site, I didn't have much experience with. Angular is definitely the least hype of the three frameworks. But, as I am not a front-end developer, I like sticking to what I know, rather than chasing the latest and greatest.

Backend

AWS Lambda NodeJS

What is Lambda?

AWS Lambda is a Amazon's option for running code, without having to operate a server. This is more commonly known as 'serverless' technology. Why this name was chosen, I'm not sure... there is still a server running your code, you just don't have to interact with it. This simplifies deploying code for developers, because they no longer need traditional operations skills. Networking, OS, security patches etc. are just no longer a concern. This works great for small dev teams, with little or no available operation resources.

Another great feature of AWS Lambda (and most serverless computing technologies), is that it is also auto-scaling. I don't need to think about how many resources to put aside for my application. AWS will handle this. To learn more about AWS Lambda, I recommend going here: https://aws.amazon.com/lambda/.

Why AWS Lambda?

So, there are several questions to this. First of all, why AWS? Well... I can't really argue for why not Azure or GCP. There are only so many different ways that you can offer running code, and honestly, they aren't all that different. I chose AWS, because, like with Angular, this is what I know and therefore it's the easiest option for me.

The second question is why serverless at all? Well, as stated above. This is ideal for small dev teams. The dev team behind this site is me... so, I feel we can quite comfortably call this a small dev team. The less the surface area of operation for me, the better. Serverless computing is the best choice for this reason.

The last question is why NodeJS for serverless? Well, personally, I would have preferred writing the backend in Golang. Perhaps, I will write the database integration in Golang at some point. But, the matter of the fact is, that the NodeJS support in AWS Lambda is spectacular. Deployment and testing are both much easier in NodeJS (in my opinion) and the performance for smaller Lambda functions is actually almost similar (proof)

...also, it's cheap :)

AWS DynamoDB Database

What is DynamoDB?

DynamoDB is a proprietary No-SQL database service provided by AWS. Much like with serverless computing this 'Database-as-a-service', basically gives you a database to work with, without having to setup a server... or a database for that matter. Basically, you just create a table via. a DynamoDB query and then you are ready to go. It's pretty impressive and more importantly, makes life for developers a lot easier.

Why DynamoDB?

It makes life easier. I like it, when life is easy. That being said, one of the typical downfalls when developing serverless applications, is overlooking the auto-scaling of lambda function, doesn't translate well, to a normal database that isn't auto-scaling. If your application get's too busy, you database will explode. This we want to avoid. With DynamoDB, we don't have to think about this, because just like lambda functions, DynamoDB offers auto-scaling, so out application will scale together with our Database.

I could also have chosen the GCP or Azure versions of this, however, keeping our application makes permissions and deployment quite a lot easier. And again... I like easy :)
